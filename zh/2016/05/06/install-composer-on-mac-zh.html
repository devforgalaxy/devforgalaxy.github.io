安裝 Composer 於 Mac OS X

May 6, 2016 PHP English

Composer 並不是單純的 PHP 套件安裝工具，它更著墨於管理套件彼此之間相依的關係，因此使用 Composer 來幫你管理 PHP 專案所需的套件會更加方便。本篇文章將介紹如何安裝與使用 Composer 於 Mac OS X 中。



安裝 Composer

我們將按照 Composer 官方的指示來安裝。其方法有兩種，分別為區域性跟全域性安裝。接下來將示範全域性安裝，首先依序執行以下指令：

接著將 Composer PHAR 檔案搬至 /usr/local/bin，指令如下：

mv composer.phar /usr/local/bin/composer

如此一來，你就可以在任何路徑下執行 composer 指令。

使用 Composer

以下為 Comopser 常用指令：

# 查詢目前版本
comoposer -version

# 查詢欲安裝的套件
comoposer search <package-name>

# 移除套件
composer remove <package-name>

此外，Composer 最方便的功能之一在於我們可以使用 composer.json 來定義我們專案所需要的套件，讓 composer 幫我們管理這些套件。composer.json 範例如下：

{
    ...
    "require": {
        "monolog/monolog": "1.0.*",
        "parse/php-sdk" : "1.0.*"
    }
    ...
}

接著，我們只需要在 composer.json 所在的路徑下執行：

# 安裝套件
composer install

# 更新套件
composer update

執行後 Composer 就會自動幫我們安裝 monolog 與 parse 套件。若你想要額外手動加入套件至專案以及 composer.json 中，你可以使用指令：

composer require <package-name>
